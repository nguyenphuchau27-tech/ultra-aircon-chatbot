==================================================
BƯỚC 2 — CUSTOMER AUTH FLOW
==================================================

Mục tiêu:
- Customer login thật với backend
- Chỉ role customer được vào Customer App
- Access token / refresh token được lưu local
- Auto attach Authorization header
- Cold start còn session thì vào app
- Refresh token hoạt động
- Refresh fail thì clear session và quay về login
- Logout xóa sạch session

Không dùng account admin/technician để test customer app.

Checklist test:
1. Login customer thành công
2. Login sai password báo lỗi rõ
3. Login role technician/admin bị chặn
4. Kill app mở lại vẫn giữ session
5. Revoke auth_sessions trong DB -> app xử lý expired session đúng
6. Logout -> quay về login, mở lại vẫn ở login

==================================================
BƯỚC 3 — CREATE ORDER FLOW
==================================================

Mục tiêu:
Customer tạo order thật qua backend:

POST /api/orders

Yêu cầu:
- Form tạo order tối thiểu:
  serviceType
  address
  lat/lng nếu backend yêu cầu hoặc lấy từ location nếu đã có
- customerId phải lấy từ session/user hiện tại, không hardcode sai
- Không cho customer tạo order thay user khác
- Loading/error/success rõ ràng
- Sau khi tạo xong phải thấy order trong danh sách

Checklist test:
1. Customer login
2. Tạo order mới
3. Backend trả 201/200
4. DB có order mới đúng customerId
5. Status ban đầu đúng: pending
6. Customer app refresh thấy order mới

==================================================
BƯỚC 4 — MY ORDERS LIST
==================================================

Mục tiêu:
Customer xem danh sách order của chính mình:

GET /api/orders/customer/:customerId

Yêu cầu:
- Không dùng /orders toàn hệ thống cho customer app nếu không cần
- Không show order của customer khác
- Hiển thị tối thiểu:
  id
  serviceType
  address
  status
  technician nếu đã assign
  createdAt
- Loading / empty / error rõ ràng

Checklist test:
1. Customer A chỉ thấy order của Customer A
2. Không thấy order của technician/admin
3. Order pending/assigned/in_progress/completed hiển thị đúng

==================================================
BƯỚC 5 — ORDER DETAIL
==================================================

Mục tiêu:
Customer mở order detail thật:

GET /api/orders/:id

Yêu cầu:
- UI detail production, không raw JSON
- Hiển thị:
  serviceType
  address
  status
  customer info
  technician info nếu có
  lat/lng nếu có
  createdAt/updatedAt
- Response không lộ password/hash

Checklist test:
1. Bấm order trong list
2. Mở detail đúng order
3. Không crash nếu technician null
4. Không hiện password/passwordHash/refreshTokenHash

Nếu có realtime/socket cho customer thì dùng thật.
Nếu chưa có, dùng refresh thủ công và ghi rõ deferred realtime customer notification phase sau.
