THỨ TỰ THỰC HIỆN BẮT BUỘC
=========================

BƯỚC 1 — AUDIT TRƯỚC KHI SỬA

Trước tiên, em phải đọc kỹ và đối chiếu các phần sau trong source hiện tại:

* backend auth flow
* backend technicians endpoints
* backend orders endpoints cho technician
* backend response DTO / serializer / mapper
* flutter api_service
* flutter login screen / dashboard / tracking / job-related screen nếu có
* admin global_map đang consume dữ liệu gì

Sau đó phải báo lại cho anh theo format:

1. Những gì Phase 5 đã DONE thật
2. Những gì còn thiếu
3. Những bug / risk đang tồn tại
4. Kế hoạch sửa theo thứ tự ưu tiên

Chưa được code ngay nếu chưa audit xong.

==================================================
BƯỚC 2 — FIX SECURITY RESPONSE
==============================

Ưu tiên cao nhất trước khi mở rộng feature.

Mục tiêu:

* mọi response từ backend trả user / technician / auth payload không được chứa password hoặc password hash

Em phải:

* tìm chính xác endpoint nào đang leak
* chỉ ra root cause nằm ở DTO / entity return / mapper / serializer / controller / service
* sửa đúng tầng
* rà lại toàn bộ endpoint liên quan auth / users / technicians

Sau khi sửa xong phải đưa checklist test:

* endpoint nào test
* request nào gọi
* kỳ vọng response không còn password hash

==================================================
BƯỚC 3 — HOÀN THIỆN JOB FLOW CHO TECHNICIAN APP
===============================================

Mục tiêu:
Technician app phải bám backend thật cho:

* job list
* job detail

Nếu source hiện tại chưa có screen hoàn chỉnh, em phải:

* tận dụng cấu trúc hiện có
* mở rộng đúng style app hiện tại
* không làm UI rườm rà
* ưu tiên đúng dữ liệu thật

Yêu cầu tối thiểu:

* dashboard load danh sách job thật hoặc summary thật
* bấm vào job xem được detail thật
* dữ liệu map đúng backend contract
* xử lý loading / empty / error rõ ràng
* nếu token hết hạn thì app xử lý đúng flow auth hiện có

Sau khi sửa xong phải đưa checklist test thật:

1. login technician
2. load job list
3. mở job detail
4. verify dữ liệu khớp backend

==================================================
BƯỚC 4 — CHỐT AUTH FLOW MOBILE CHO SẠCH
=======================================

Rà lại toàn bộ auth flow technician app:

* lưu access token
* auto attach Authorization header
* refresh token flow
* cold start còn session thì vào dashboard
* session hết hạn thì quay về login đúng cách
* logout xóa sạch session local

Nếu phần này đã có rồi, em phải verify lại và chỉ sửa nếu có bug thật.

Checklist test bắt buộc:

1. login thành công
2. kill app / mở lại app
3. token còn hạn -> vào dashboard
4. xóa auth_sessions trong DB hoặc làm expire session
5. app phải xử lý đúng: refresh được thì tiếp tục, refresh fail thì về login

==================================================

Mục tiêu:
có bộ seed/dev data sạch, dùng ổn định cho test Phase 5.

Bắt buộc có đủ:

* admin
* customer
* technician

Cần làm rõ:

* email
* password
* role
* status
* technician profile nếu role technician cần mapping riêng
* dữ liệu order/job mẫu để job list / job detail test được thật

Nếu chưa có script seed chuẩn:

* viết hoặc chỉnh lại seed theo source hiện tại
* tránh dữ liệu trùng / bẩn / role sai / hash sai

Phải trả lại cho anh:

1. danh sách tài khoản test chuẩn
2. password test chuẩn
3. dữ liệu order/job mẫu để test
4. cách reset seed nếu cần
