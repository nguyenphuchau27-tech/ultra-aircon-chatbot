========================
DEBUG PROTOCOL (BẮT BUỘC TUÂN THỦ)
========================

Mọi bug hoặc task đều phải xử lý theo quy trình này:

STEP 1 — XÁC ĐỊNH HIỆN TƯỢNG
- Mô tả chính xác lỗi hoặc phần chưa done:
  + xảy ra ở đâu (UI / API / socket / DB / auth flow)
  + hành vi hiện tại là gì
  + expected behavior là gì
- KHÔNG được suy đoán nguyên nhân ở bước này

STEP 2 — KHOANH VÙNG NGUYÊN NHÂN
- Xác định layer gây lỗi:
  + frontend
  + API layer
  + backend
  + realtime/socket
  + database
- Chỉ rõ file liên quan

STEP 3 — XÁC NHẬN ROOT CAUSE
- Đọc code thật
- Chỉ ra dòng/đoạn logic gây lỗi
- Giải thích vì sao nó gây lỗi
- Nếu chưa đủ bằng chứng:
  → yêu cầu thêm log hoặc output
  → KHÔNG được sửa

STEP 4 — ĐỀ XUẤT FIX (MINIMAL)
- Chỉ sửa đúng phần gây lỗi
- Không rewrite lan rộng
- Không sửa nhiều file không cần thiết

STEP 5 — VIẾT CODE
- Viết FULL FILE để copy
- Code phải production-ready
- Không workaround tạm
- Không thêm logic dư thừa

STEP 6 — VERIFY
- Hướng dẫn test lại
- Nêu expected result
- Chỉ ra cách xác nhận đã done

STEP 7 — KHÔNG LÀM
- Không đoán mò
- Không sửa khi chưa xác định root cause
- Không sửa lan rộng nhiều file
- Không refactor khi đang debug
- Không thay đổi cấu trúc project
